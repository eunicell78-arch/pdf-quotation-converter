# 🐛 문제 리포트 가이드 - 변환 결과가 잘못된 경우

## 변환 결과가 이상한가요?

변환은 되지만 데이터가 엉뚱하게 들어가 있나요? 이 가이드를 따라 문제를 보고해주세요!

---

## 📸 1단계: 변환 결과 공유하기

### 방법 A: CSV 파일 스크린샷

1. **CSV 파일 열기**
   - 다운로드된 CSV 파일을 Excel, Google Sheets, 또는 메모장으로 엽니다

2. **스크린샷 찍기**
   - Windows: `Win + Shift + S` 또는 `Print Screen`
   - Mac: `Cmd + Shift + 4`
   - 파일의 처음 10-20줄 정도를 캡처

3. **파일로 저장**
   - PNG 또는 JPG로 저장

### 방법 B: CSV 내용 텍스트로 복사

1. CSV 파일을 메모장으로 열기
2. 처음 10-20줄 복사
3. GitHub Issue 또는 댓글에 붙여넣기

**예시:**
```csv
Date,Customer,Planner,Product,Rated Current,Cable Length,...
2024-01-15,ABC Company,John,Product A,10A,2m,...
```

---

## 🔍 2단계: 브라우저 콘솔 로그 공유하기

웹 버전을 사용 중이라면 개발자 콘솔의 로그가 매우 유용합니다!

### 콘솔 열기

**Chrome/Edge:**
1. `F12` 키 누르기 (또는 우클릭 → "검사")
2. **"Console"** 탭 선택

**Firefox:**
1. `F12` 키 누르기
2. **"콘솔"** 탭 선택

**Safari:**
1. 환경설정 → 고급 → "메뉴 막대에서 개발자용 메뉴 보기" 체크
2. 개발자 메뉴 → JavaScript 콘솔 표시

### 로그 내보내기

1. 콘솔에서 모든 메시지를 선택 (Ctrl+A 또는 Cmd+A)
2. 복사 (Ctrl+C 또는 Cmd+C)
3. 텍스트 파일에 붙여넣기
4. GitHub Issue에 첨부

**중요한 로그:**
```
Detected columns: {...}
Extracted items: [...]
Total items extracted: 5
```

이 로그들이 어떻게 표시되는지 공유해주세요!

---

## 📄 3단계: 원본 PDF 정보 공유

### 공유 가능한 정보:

1. **PDF 파일명** (개인정보 제거)
2. **PDF 구조 설명**:
   - 테이블이 몇 페이지에 있나요?
   - 열 이름은 무엇인가요? (Item, Product, MOQ, Price 등)
   - 병합된 셀이 있나요?

3. **스크린샷** (민감한 정보는 가리고):
   - PDF의 테이블 구조
   - 헤더 행
   - 데이터 행 2-3개

### ⚠️ 개인정보 보호

**제거해야 할 정보:**
- 회사명, 고객명
- 가격, 금액
- 연락처, 이메일
- 기타 민감한 정보

**방법:**
- 이미지 편집 프로그램으로 검은색 박스로 가리기
- 또는 "Company A", "Product X" 같은 가명 사용

## 🎯 4단계: 문제 설명

### 좋은 문제 보고 예시:

```
제목: CSV 변환 시 Product 열에 MOQ 데이터가 들어갑니다

증상:
- PDF에서 Product 열에 있던 제품명이 CSV의 MOQ 열에 들어감
- MOQ 데이터는 Price 열에 들어감
- 모든 데이터가 한 칸씩 밀린 것 같음

환경:
- 브라우저: Chrome 120
- OS: Windows 11
- PDF 페이지 수: 3페이지

첨부:
- CSV 결과 스크린샷
- 브라우저 콘솔 로그
- PDF 테이블 구조 스크린샷 (개인정보 제거)
```

---

## 🔧 5단계: 임시 해결 방법

### 데스크톱 버전 사용

웹 버전이 제대로 작동하지 않으면 데스크톱 버전을 시도해보세요:

```bash
# Python 설치 후
pip install -r requirements.txt

# CLI 버전
python converter.py input.pdf output.csv

# 또는 GUI 버전
python converter_gui.py
```

데스크톱 버전은 더 강력한 PDF 파싱 라이브러리(pdfplumber)를 사용합니다.

## 📊 6단계: 콘솔 로그 상세 분석

### 정상적인 로그 예시:

```javascript
// 1. 헤더 감지
Detected columns: {
  item: {index: 0, x: 50, text: "Item"},
  product: {index: 1, x: 100, text: "Product"},
  moq: {index: 3, x: 300, text: "MOQ"},
  ...
}

// 2. 데이터 추출
Extracted items: [
  {
    item: "1",
    product: "Cable Assembly",
    moq: "1000",
    price: "$1.50",
    ...
  },
  ...
]

// 3. 요약
Total items extracted: 5
```

### 문제가 있는 로그 예시:

```javascript
// 헤더를 못 찾음
No table header found with Product and MOQ columns

// 또는 잘못된 열 감지
Detected columns: {
  item: null,  // ← 문제!
  product: {index: 0, x: 50, text: "Something else"},
  ...
}

// 또는 데이터가 비어있음
Extracted items: []
```

이런 로그가 보이면 함께 공유해주세요!

## 🎓 자주 하는 실수

### 1. 브라우저 캐시 문제
**증상**: 이전 버전의 코드가 실행됨
**해결**: `Ctrl+Shift+R` (강제 새로고침)

### 2. PDF 형식 문제
**증상**: 스캔 이미지 PDF는 변환 불가
**해결**: 텍스트 기반 PDF인지 확인 (PDF에서 텍스트 선택 가능한지 테스트)

### 3. 복잡한 테이블 구조
**증상**: 여러 단계로 병합된 셀이 있는 경우
**해결**: 데스크톱 버전 사용
