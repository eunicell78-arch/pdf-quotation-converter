# 📎 파일 공유 방법 - 개발자가 직접 확인하고 수정

## 파일을 공유하는 3가지 방법

### 🎯 방법 1: GitHub Issue 생성 (가장 좋음!)

1. **GitHub Issues 페이지로 이동**
   - https://github.com/eunicell78-arch/pdf-quotation-converter/issues

2. **"New Issue" 버튼 클릭**

3. **제목 작성**
   ```
   [웹 변환] 데이터가 잘못 변환됨 - 열 정렬 오류
   ```

4. **파일 첨부**
   - PDF 파일을 드래그 앤 드롭 (또는 클릭하여 선택)
   - CSV 파일도 첨부
   - 스크린샷도 함께 첨부 가능

5. **내용 작성 (아래 템플릿 사용)**

```markdown
## 문제 설명
변환 결과가 이상합니다. Product 열에 MOQ 데이터가 들어가고, 모든 열이 한 칸씩 밀린 것 같습니다.

## 첨부 파일
- [ ] 원본 PDF 파일 (개인정보 제거 또는 샘플)
- [ ] 변환된 CSV 파일
- [ ] 브라우저 콘솔 로그 (아래 참조)

## 브라우저 콘솔 로그
```
[여기에 F12 → Console 탭의 로그 붙여넣기]
```

## 환경
- 브라우저: Chrome 120 (버전 확인: chrome://version)
- OS: Windows 11

## 기대했던 결과
각 열에 올바른 데이터가 들어가야 합니다.

## 실제 결과
열이 한 칸씩 밀려서 잘못된 위치에 데이터가 들어갔습니다.
```

---

### 📋 방법 2: CSV 내용 텍스트로 붙여넣기

파일을 직접 첨부할 수 없다면:

1. **CSV 파일을 메모장으로 열기**
   - 우클릭 → "연결 프로그램" → "메모장"

2. **처음 20줄 정도 복사**

3. **GitHub Issue 또는 댓글에 붙여넣기**
   - \`\`\` 로 감싸서 코드 블록으로 작성

**예시:**
```
Date,Customer,Planner,Product,Rated Current,Cable Length,Description,Delivery Term,MOQ,Price,L/T,Remark
2024-01-15,ABC Corp,John,Cable A,10A,2m,Description,FOB,1000,1.50,4,Sample
2024-01-15,ABC Corp,John,Cable B,15A,3m,Description,FOB,2000,2.00,6,
```

4. **PDF 구조도 설명**
   - "PDF에는 Item, Product, Delivery Term, MOQ, Unit Price, L/T, Remark 열이 있습니다"
   - "데이터는 3페이지에 걸쳐 있습니다"

---

### 🖼️ 방법 3: 스크린샷 공유

민감한 정보가 있어서 파일을 직접 공유하기 어렵다면:

#### A. CSV 결과 스크린샷

1. CSV를 Excel이나 스프레드시트로 열기
2. 처음 10-20행 스크린샷
3. GitHub Issue에 이미지 드래그 앤 드롭

#### B. PDF 구조 스크린샷

1. PDF의 테이블 부분 스크린샷
2. **민감한 정보 가리기**:
   - 그림판이나 이미지 편집기로 검은 박스 그리기
   - 또는 Snipping Tool로 선택적으로 캡처

3. 헤더 행과 데이터 행 2-3개만 보이도록

**가려야 할 정보:**
- ❌ 회사명, 고객명
- ❌ 실제 가격, 금액
- ❌ 연락처
- ✅ 열 이름 (Item, Product, MOQ 등)
- ✅ 테이블 구조
- ✅ 샘플 데이터 (가명 사용)

## 💡 임시 해결책

파일을 공유하기 전에 시도해볼 수 있는 것들:

### 1. 브라우저 캐시 강제 새로고침
```
Ctrl + Shift + R (Windows)
Cmd + Shift + R (Mac)
```

### 2. 다른 브라우저에서 시도
- Chrome ✅ (가장 잘 작동)
- Edge ✅
- Firefox ⚠️
- Safari ⚠️

### 3. 데스크톱 버전 사용
```bash
pip install -r requirements.txt
python converter.py your-file.pdf output.csv
```

데스크톱 버전은 pdfplumber를 사용하여 더 정확합니다.
