# ✅ Streamlit Cloud 배포 체크리스트

웹 배포 전에 이 체크리스트를 확인하세요!

## 📋 배포 전 체크리스트

### 1. GitHub 저장소 확인
- [ ] 코드가 GitHub에 push되어 있음
- [ ] 저장소가 public 또는 Streamlit Cloud에서 접근 가능
- [ ] Branch 이름 확인: `copilot/add-gui-for-pdf-to-csv`

### 2. 필수 파일 확인
- [x] `streamlit_app.py` - 메인 앱 파일 ✅
- [x] `requirements.txt` - Python 패키지 목록 ✅
- [x] `packages.txt` - 시스템 패키지 목록 (poppler-utils) ✅
- [x] `.streamlit/config.toml` - Streamlit 설정 ✅
- [x] `converter.py` - 변환 로직 ✅

### 3. Dependencies 확인
- [x] `requirements.txt`에 필요한 패키지:
  - `pdfplumber>=0.9.0` ✅
  - `pandas>=2.0.0` ✅
  - `openpyxl>=3.1.0` ✅
  - `streamlit>=1.29.0` ✅

### 4. 시스템 패키지 확인
- [x] `packages.txt`에 `poppler-utils` 포함 ✅

### 5. 코드 검증
- [x] 로컬에서 실행 테스트 완료 ✅
- [x] 샘플 PDF로 변환 테스트 완료 ✅
- [x] 에러 핸들링 구현 완료 ✅

## 🚀 배포 단계

### Step 1: Streamlit Cloud 접속
```
https://share.streamlit.io
```

### Step 2: New app 생성
- "New app" 버튼 클릭

### Step 3: 정보 입력
```
Repository: eunicell78-arch/pdf-quotation-converter
Branch: copilot/add-gui-for-pdf-to-csv  
Main file path: streamlit_app.py
```

### Step 4: 배포!
- "Deploy!" 버튼 클릭
- 2-3분 대기

### Step 5: 완료 확인
- [ ] 앱이 정상적으로 로드됨
- [ ] PDF 업로드 테스트
- [ ] 변환 기능 테스트
- [ ] CSV 다운로드 테스트
- [ ] Excel 다운로드 테스트

## 🐛 문제 발생 시

### 빌드 실패
1. Streamlit Cloud 로그 확인
2. `requirements.txt` 형식 확인
3. Python 버전 확인 (3.8-3.11)

### 앱 실행 안 됨
1. 2-3분 더 대기
2. 브라우저 새로고침
3. "Reboot app" 클릭

### PDF 변환 실패
1. `packages.txt` 확인
2. 로그에서 에러 메시지 확인
3. 앱 재부팅
