# 🚨 404 오류 해결 가이드

## 문제: 웹 링크에 접속하면 404 오류 발생

`https://eunicell78-arch.github.io/pdf-quotation-converter/` 링크가 작동하지 않나요?

이는 **GitHub Pages가 아직 활성화되지 않았기 때문**입니다.

---

## ✅ 해결 방법

### 👨‍💼 저장소 관리자인 경우:

#### 단계 1: GitHub Pages 활성화

1. **GitHub 저장소 접속**
   - https://github.com/eunicell78-arch/pdf-quotation-converter

2. **Settings → Pages**
   - 저장소 상단의 "Settings" 탭 클릭
   - 왼쪽 메뉴에서 "Pages" 클릭

3. **Source 설정**
   - **Branch** 드롭다운에서 다음 중 하나 선택:
     - `copilot/add-gui-for-pdf-to-csv` (추천)
     - 또는 `gh-pages` (아직 없다면 생성 필요)
   - **Folder**: `/ (root)` 선택
   - **Save** 버튼 클릭

4. **배포 대기**
   - 첫 배포는 1-5분 소요
   - 페이지 상단에 배포 URL이 표시됨
   - Actions 탭에서 배포 진행 상황 확인 가능

5. **테스트**
   - 표시된 URL 클릭: `https://eunicell78-arch.github.io/pdf-quotation-converter/`
   - 웹 변환기가 정상적으로 로드되는지 확인

#### 단계 2: gh-pages 브랜치 생성 (선택사항)

더 안정적인 배포를 위해 `gh-pages` 브랜치를 사용할 수 있습니다:

```bash
# 저장소 클론
git clone https://github.com/eunicell78-arch/pdf-quotation-converter.git
cd pdf-quotation-converter

# gh-pages 브랜치 생성 및 체크아웃
git checkout copilot/add-gui-for-pdf-to-csv
git checkout -b gh-pages

# gh-pages 브랜치 푸시
git push origin gh-pages
```

GitHub이 자동으로 `gh-pages` 브랜치를 감지하여 배포합니다.

## 🔄 대체 방법 (GitHub Pages 없이 사용)

### 방법 1: ZIP 다운로드 후 로컬 실행 ⭐ (추천)

1. **ZIP 파일 다운로드**
   - [여기를 클릭](https://github.com/eunicell78-arch/pdf-quotation-converter/archive/refs/heads/copilot/add-gui-for-pdf-to-csv.zip)

2. **압축 해제**
   - 다운로드한 ZIP 파일을 원하는 위치에 압축 해제

3. **로컬 웹 서버 실행**
   ```bash
   # 압축 해제한 폴더로 이동
   cd pdf-quotation-converter
   
   # Python 웹 서버 실행
   python -m http.server 8000
   ```

4. **브라우저에서 접속**
   - http://localhost:8000

#### Python이 없는 경우:

**Node.js 사용:**
```bash
npx http-server -p 8000
```

**VS Code 사용:**
- VS Code에서 폴더 열기
- "Live Server" 확장 설치
- index.html 우클릭 → "Open with Live Server"

### 방법 2: 파일로 직접 열기

1. ZIP 다운로드 및 압축 해제
2. `index.html` 파일을 더블클릭하거나 브라우저로 드래그

⚠️ **주의**: 
- 일부 브라우저는 보안상 이유로 로컬 파일에서 외부 리소스(PDF.js CDN) 로딩을 차단할 수 있습니다
- Chrome/Edge에서 "CORS 오류"가 발생하면 위의 로컬 서버 방법을 사용하세요

### 방법 3: 데스크톱 버전 사용

웹 버전 대신 Python 기반 GUI 프로그램을 사용할 수 있습니다:

1. **저장소 클론 또는 ZIP 다운로드**
2. **Python 설치** (https://python.org)
3. **의존성 설치**
   ```bash
   pip install -r requirements.txt
   ```
4. **GUI 프로그램 실행**
   - Windows: `run_converter.bat` 더블클릭
   - 또는: `python converter_gui.py`

자세한 내용: [다운로드_방법.md](다운로드_방법.md)

## ❓ 자주 묻는 질문

### Q1: 왜 404 오류가 발생하나요?
**A**: GitHub Pages가 활성화되지 않았거나, 잘못된 브랜치가 선택되었기 때문입니다.

### Q2: GitHub Pages 활성화에 얼마나 걸리나요?
**A**: 첫 배포는 1-5분, 이후 업데이트는 1-2분 정도 소요됩니다.

### Q3: Private 저장소에서도 GitHub Pages를 사용할 수 있나요?
**A**: GitHub Pro 플랜 이상에서 가능합니다. 무료 플랜은 Public 저장소만 지원합니다.

### Q4: 브랜치를 선택했는데도 404가 나옵니다.
**A**: 
1. Actions 탭에서 배포 진행 상황 확인
2. 배포가 완료될 때까지 기다림 (최대 5분)
3. 브라우저 캐시 삭제 후 재시도
4. Settings → Pages에서 올바른 URL 확인

### Q5: 로컬에서 실행하면 PDF.js 오류가 발생합니다.
**A**: CORS 정책 때문입니다. `python -m http.server` 같은 로컬 웹 서버를 사용하세요.
